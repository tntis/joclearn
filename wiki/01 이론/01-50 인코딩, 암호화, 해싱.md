# 인코딩, 암호화, 해싱

## 인코딩(Encoding)
- 주어진 입력에 대한 모든 변환을 의미
- 문자열을 뒤집는 함수 `x`가 있을 때, `x → y`를 `ABCD`에 적용하면, `DCBA`가 나옴

<br/>


## 암호화(Encryption)
- 출력을 얻기 위해 입력 값과 키를 모두 지정하는 특정한 유형의 인코딩
- 키를 이용해 “출력 → 입력”(역방향)을 얻는 함수를 호출할 수 있는지 선택 가능
- 암호화를 함수로 나타낸 가장 간단한 형식 : `(x, k) → y`
    - `x`는 입력, `k`는 키, `y`는 암호화 결과
- **역함수 복호화(Reverse Function Decryption)** : `(y, k) → x`
    - 알려진 함수에 키를 이용해 출력에서 입력을 얻는 방식
    - **대칭 키** : 암호화에 쓰이는 키와 복호화에 쓰이는 키가 같은 경우
        - 암호화 : `(x, k) → y`
        - 복호화 : `(y, k) → x`
    - **비대칭 키(Asymmetric Key)** : 암호화에 쓰이는 키와 복호화에 쓰이는 키가 다른 경우
        - 암호화 : `(x, k1) → y`
        - 복호화 : `(y, k2) → x`
        - `(k1, k2)` : **키 쌍(Key Pair)**
            - 암호화에 이용되는 키 `k1` : **공개 키(Public Key)**
            - 복호화에 이용되는 키 `k2` : **개인 키(Private Key)**; 개인 키 소유자는 데이터 복호화 가능

### 비대칭 키 용도 - 참고

- **암호화/복호화**
    - 암호화: 공개 키 사용 (누구나 암호화 가능)
    - 복호화: 개인 키 사용 (개인 키 소유자만 가능)
- **전자서명/검증**
    - 서명: 개인 키 사용 (개인 키 소유자만 가능)
    - 검증: 공개 키 사용 (누구나 검증 가능)

## 해싱(Hashing)
- 함수가 한 방향으로만 작동하는 특정한 유형의 인코딩
- 해싱 함수의 출력 `y`에서 입력 `x`를 얻을 수 없음
- 그러나, 출력 `y`가 입력 `x`에 해당되는지 확인할 수 있는 방법이 반드시 있어야 함
- 해시는 한쌍의 인코딩 함수와 일치 함수로 볼 수 있음
- 해싱 함수가 `x → y`라면, 일치 함수 `(x, y) → boolean`도 있음
- 해싱 함수는 입력에 임의의 값 추가 가능
    - `(x, k) → y` : `k`는 **솔트(Salt)**
    - 솔트는 함수를 더 강하게 만들어 결과에서 입력을 얻는 역함수의 적용 난도를 높임
